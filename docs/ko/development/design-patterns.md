# 구현 설계 패턴

이 가이드는 현재 Rust 워크스페이스에서 보이는 안정적인 구현 구조를
설명합니다. 개발자 학습 문서이므로 문제, 소스 위치, 장점, 트레이드오프,
테스트를 이름으로 가리키지만 공개 API 동작, 스키마, 저장 효과, 보안
보장, 런타임 동작, Core 권한 의미를 정의하지 않습니다.

여기서 "닮았다", "기능한다", "여기서는 ...로 쓰인다" 같은 표현을
의도적으로 사용합니다. 소스는 오래 유지되는 구조를 보여 주지만, 이
페이지는 코드가 보여 주는 것 이상의 공식 패턴 의도를 주장하지 않습니다.

정확한 제품 동작은 [참조 색인](../reference/README.md)을 사용합니다.
넓은 크레이트 지도는 [구현 아키텍처](architecture.md)를, 대표 메서드
흐름은 [요청 생명주기](request-lifecycle.md)를 사용합니다.

## 이 페이지를 사용하는 법

좁은 변경을 수행할 때 이 페이지로 어떤 구현 구조를 건드리는지 확인합니다.
그다음 [구현 가이드](change-guide.md)에서 정확한 참조 담당 문서와 테스트
계층을 찾습니다. 어떤 동작이 패턴에 "맞아 보인다"는 이유로 새 동작을
추가하지 않습니다. 적용되는 담당 문서가 정의했거나 담당 문서 공백이
해결되었을 때만 동작을 추가합니다.
