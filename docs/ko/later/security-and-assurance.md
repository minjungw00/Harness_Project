# 이후 후보: 보안과 보증

## 이 문서가 담당하는 것

이 문서는 더 강한 보안 태세, 보증 주장, 역량 라벨, 관찰, 차단, 검증 신뢰도에 관한 비활성 이후 후보를 담당합니다. 후보를 이 문서에 모아 [이후 후보 색인](index.md)이 경로 안내와 짧은 요약으로 남을 수 있게 합니다.

여기의 모든 후보는 향후 가능성입니다. 후보 상세는 문서 원천 자료일 뿐이며 런타임 동작을 활성화하지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP 보안 보장, 현재 접근 등급, 현재 API 메서드, 저장 효과, UI 동작, 커넥터 동작, 실행 가능한 적합성, 구현 준비 상태를 정의하지 않습니다.

또한 더 강한 보장이 가능하다고 결정하지 않습니다. 예방형, 탐지형, 격리, 가림, 관찰, 차단 주장은 승격 때 현재 보안 담당 문서와 현재 범위 담당 문서가 다시 담당해야 합니다.

## Category 경계

이 카테고리는 "하네스가 어떤 보증을 정직하게 주장할 수 있는가?"가 핵심인 후보를 담습니다. 예방형 통제 후보, 격리 라벨, 역량 프로필 강화, 명령/네트워크/비밀값 관찰, 도구 실행 전 차단, 캡처 보증, 더 강한 검증 신뢰도 주장이 여기에 들어갑니다.

아티팩트 저장 방식으로서의 네이티브 캡처, 접점별 커넥터 설계, 팀 작업 흐름, 검증기 카탈로그 세부사항은 담당하지 않습니다. 단, 후보의 핵심이 보증 주장이라면 이 문서가 승격 전 보안과 보증 관점을 담당합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 보증 강화 | 현재 MVP보다 강한 증거, 검증, 닫기 준비 상태 보증 주장. |
| 더 강한 로컬 역량 프로필 | 관찰, 캡처, 격리, 차단 역량을 나타내는 향후 프로필 라벨. |
| 명령, 네트워크, 비밀값 접근 관찰 | 선택된 도구 의도나 접근 패턴을 관찰하는 향후 가능성. |
| 명령, 네트워크, 비밀값 도구 실행 전 차단 | 도구 실행 전에 예방형 통제를 주장하는 향후 가능성. |
| 역량 조건부 권한 관찰 필드 | `intended_commands`, `intended_network`, `intended_secret_scope`, `network_write`, `external_service_write`, `secret_access` 같은 향후 필드. |
| 네이티브 후크와 고급 사이드카 감시 | 더 넓은 도구 가시성을 뒷받침할 수 있는 향후 네이티브 후크나 사이드카 감시 주장. |
| 캡처 가림과 보존 보증 | 브라우저 캡처, 아티팩트 본문 캡처, 가림, 보존에 관한 향후 보안 주장. |
| 검증 보증 강화 | 분리형 검증이나 접점 간 검증을 포함한 더 강한 검증 신뢰도 주장. |

## 후보 상세

### 보증 강화

요약: 하네스가 현재 MVP보다 강한 보증 문장을 할 수 있다는 향후 주장을 추적합니다.

상태: 이후 후보.

현재 아님: 현재 MVP에 포함되지 않으며 현재 요구사항을 만들지 않습니다.

승격 조건: 승격 때 현재 범위 문서 갱신, 관련 현재 담당 문서 갱신 또는 새 담당 문서 지정, 동작이 규범이 되면 적합성/점검 갱신이 필요합니다.

현재 담당 문서 영향: 승격 전에는 없습니다.

## 승격 규칙

승격은 이 파일만 고쳐서 이루어지지 않습니다. 후보가 현재 기능이 되려면 같은 문서 전용 작업 묶음에서 현재 범위와 관련 현재 담당 문서가 함께 갱신되어야 합니다.

승격할 동작의 현재 담당 문서가 없다면, 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 그 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기의 언급은 보장, 프로필, 필드, 메서드, 접근 등급, 검증기, fixture, UI, 런타임 통제를 활성화하지 않습니다.
