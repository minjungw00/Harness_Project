# 이후 후보: 작업 흐름과 협업

## 이 문서가 담당하는 것

이 문서는 더 풍부한 작업 구체화, 다음 행동 흐름, 검증과 평가 작업 흐름, 사용자 판단 분기, 위험 검토, 릴리스 인계, reconcile, 팀 오케스트레이션, 협업 생명주기에 관한 비활성 이후 후보를 담당합니다. 후보를 이 문서에 모아 [이후 후보 색인](index.md)이 경로 안내와 짧은 요약으로 남을 수 있게 합니다.

여기의 모든 후보는 향후 가능성입니다. 후보 상세는 문서 원천 자료일 뿐이며 현재 작업 흐름, API, 저장소, UI, 협업 요구사항을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP API 동작, 작업 생명주기, 닫기 준비 상태 동작, 수락 동작, 잔여 위험 수락, 팀 권한, 릴리스 자동화, 저장 효과, 보안 보장, 적합성 동작, 구현 준비 상태를 정의하지 않습니다.

또한 이후 판단 분기를 사용자 소유 판단의 대체물로 만들지 않습니다. 민감 동작 승인, 최종 수락, 잔여 위험 수락, 면제, reconcile, 검증 위험 수락은 관련 현재 담당 문서를 승격 작업에서 명시적으로 바꾸기 전까지 서로 구분됩니다.

## Category 경계

이 카테고리는 "하네스가 나중에 사람, 에이전트, 판단, 생명주기 단계를 어떻게 조율할 수 있는가?"가 핵심인 후보를 담습니다. 구체화 기록, 다음 행동, 더 풍부한 검증과 평가 흐름, `Decision Packet` 표시, 위험 검토, reconcile, 릴리스 인계, 팀 작업 흐름, 파생 작업 맥락이 여기에 들어갑니다.

커넥터 세부 동작, 아티팩트 본문 저장, 보안 보장, 검증기 카탈로그는 담당하지 않습니다. 향후 작업 흐름이 그런 영역에 의존하더라도 이 문서는 승격 전 작업 흐름과 협업 관점만 기록합니다.

| 후보 | 요약 |

| 검증, Eval, 수동 QA 작업 흐름 API | 향후 `harness.launch_verify`, `harness.record_eval`, `harness.record_manual_qa`와 관련 흐름 분기. |
| 전체 `Decision Packet`과 `presentation=full` | 향후 전체 형식 판단 표시. |
| 상세 위험 검토와 잔여 위험 생명주기 | 향후 더 풍부한 위험 검토 자료, 잔여 위험 생명주기, 만료 동작. |
| 이후 사용자 판단 분기 | 향후 `qa_waiver`, `verification_risk_acceptance`, waiver, reconcile, residual-risk, 더 풍부한 수락 분기. |
| 복구와 reconcile | 향후 복구, reconcile, 상태 복구 조율 작업 흐름. |
| 릴리스 인계와 배포 자동화 | 향후 인계, 릴리스, 배포, 카나리, 롤백, 병합, 모니터링 작업 흐름. |
| 팀 작업 흐름과 오케스트레이션 | 향후 팀 권한, 공유 역량 집합, 오케스트레이션, 병렬 흐름 동작. |
| Context Index와 파생 지표 | 스스로 권한이 되지 않으면서 작업 흐름 검토를 돕는 향후 맥락과 파생 지표. |

## 후보 상세

### Discovery Brief, Question Queue, Assumption Register

요약: 열린 질문, 가정, 작업 맥락을 담는 향후 구체화 기록을 담습니다.

상태: 이후 후보.

현재 아님: 현재 MVP에 포함되지 않으며 현재 요구사항을 만들지 않습니다.

승격 조건: 승격 때 현재 범위 문서 갱신, 관련 현재 담당 문서 갱신 또는 새 담당 문서 지정, 동작이 규범이 되면 적합성/점검 갱신이 필요합니다.

현재 담당 문서 영향: 승격 전에는 없습니다.

## 승격 규칙

승격은 이 파일만 고쳐서 이루어지지 않습니다. 후보가 현재 기능이 되려면 같은 문서 전용 작업 묶음에서 현재 범위와 관련 현재 담당 문서가 함께 갱신되어야 합니다.

승격할 동작의 현재 담당 문서가 없다면, 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 그 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기의 언급은 작업 흐름 분기, API 메서드, 상태 전이, 판단 경로, 권한 모델, 릴리스 자동화, 팀 동작, UI, 저장 효과, 닫기 준비 상태 요구사항을 활성화하지 않습니다.
