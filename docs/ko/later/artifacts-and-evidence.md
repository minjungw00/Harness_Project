# 이후 후보: 아티팩트와 증거

## 이 문서가 담당하는 것

이 문서는 더 풍부한 증거, 네이티브 아티팩트 캡처, 아티팩트 본문 정책, 내보낼 수 있는 증거 묶음, 증거 관련 표시나 템플릿 확장에 관한 비활성 이후 후보를 담당합니다. 후보를 이 문서에 모아 [이후 후보 색인](index.md)이 경로 안내와 짧은 요약으로 남을 수 있게 합니다.

여기의 모든 후보는 향후 가능성입니다. 후보 상세는 문서 원천 자료일 뿐이며 현재 증거, 아티팩트, 저장소, 닫기 준비 상태 동작을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP 아티팩트 저장 동작, 현재 아티팩트 본문 읽기, 증거 충분성, 현재 API 메서드, 저장 효과, UI 동작, 보안 보장, 적합성 요구사항을 정의하지 않습니다.

또한 아티팩트 캡처를 증거 수락과 같은 것으로 만들지 않습니다. 승격 때는 Core 소유 상태, 아티팩트, 사용자 소유 판단, 검증, 수락, 잔여 위험의 경계를 보존해야 합니다.

## Category 경계

이 카테고리는 "하네스가 나중에 어떤 더 풍부한 자료를 기록, 참조, 포장, 요약할 수 있는가?"가 핵심인 후보를 담습니다. 네이티브 캡처, 전체 `Evidence Manifest`, 내보내기 묶음, 아티팩트 본문 정책, 브라우저 캡처 증거, 증거 관련 템플릿 이름이 여기에 들어갑니다.

보안 통제 주장, 커넥터 세부 동작, 검증기 카탈로그, 협업 생명주기는 담당하지 않습니다. 그런 주제가 필요하더라도 이 문서는 승격 전 아티팩트와 증거 자료 관점만 기록합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 전체 `Evidence Manifest` | 향후 매니페스트 수준 증거 기록과 표시 후보. |
| 네이티브 아티팩트 캡처 | 참조 전용 아티팩트 기록을 넘어 아티팩트 본문을 캡처하는 향후 후보. |
| 아티팩트 본문 정책 | 보존, 읽기 자격, 가림, 체크섬, 크기 정책에 관한 향후 후보. |
| 내보내기와 인계 묶음 | 증거, 아티팩트, 출처 추적, 인계 패키지 형식에 관한 향후 후보. |
| 브라우저 캡처 증거 | 브라우저 스크린샷, 녹화, 캡처된 UI 상태를 증거 자료로 다루는 향후 후보. |
| 이후 참조와 아티팩트 값 계열 | 번들, 매니페스트, QA 캡처, 내보내기 구성요소, 설계, Eval, 수동 QA, TDD, `Projection`, 관련 참조의 향후 값 계열. |
| 증거 관련 템플릿 이름 | `EVIDENCE-MANIFEST`, `EXPORT`, `RUN-SUMMARY`와 관련 표시 본문 같은 향후 템플릿 이름. |
| 읽기 전용 맥락 메타데이터 | 판단을 그 자체로 충족하지 않으면서 증거 검토를 도울 수 있는 향후 플레이북과 판단 맥락 메타데이터. |

## 후보 상세

### 전체 `Evidence Manifest`

요약: 향후 매니페스트 수준 증거 기록과 렌더링된 요약을 담습니다.

상태: 이후 후보.

현재 아님: 현재 MVP에 포함되지 않으며 현재 요구사항을 만들지 않습니다.

승격 조건: 승격 때 현재 범위 문서 갱신, 관련 현재 담당 문서 갱신 또는 새 담당 문서 지정, 동작이 규범이 되면 적합성/점검 갱신이 필요합니다.

현재 담당 문서 영향: 승격 전에는 없습니다.

## 승격 규칙

승격은 이 파일만 고쳐서 이루어지지 않습니다. 후보가 현재 기능이 되려면 같은 문서 전용 작업 묶음에서 현재 범위와 관련 현재 담당 문서가 함께 갱신되어야 합니다.

승격할 동작의 현재 담당 문서가 없다면, 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 그 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기의 언급은 아티팩트 캡처, 증거 수락, 본문 저장, 내보내기 형식, 템플릿 본문, UI 카드, 적합성 기대를 활성화하지 않습니다.
