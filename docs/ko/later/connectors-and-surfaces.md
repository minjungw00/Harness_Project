# 이후 후보: 커넥터와 접점

## 이 문서가 담당하는 것

이 문서는 향후 IDE, CLI, 채팅, MCP, 호스팅, 원격, 대시보드, 커넥터 대상 접점에 관한 비활성 이후 후보를 담당합니다. 후보를 이 문서에 모아 [이후 후보 색인](index.md)이 경로 안내와 짧은 요약으로 남을 수 있게 합니다.

여기의 모든 후보는 향후 가능성입니다. 후보 상세는 문서 원천 자료일 뿐이며 현재 접점 지원, 커넥터 권한, 호스팅 런타임 동작, UI 요구사항을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP API 메서드, 보안 보장, 아티팩트 본문 정책, 검증기 카탈로그, 적합성 fixture, 호스팅 서비스, 원격 런타임 동작, 구현 준비 상태를 정의하지 않습니다.

또한 `surface_id`, 커넥터 이름, 대시보드, 호스팅 작업 흐름, 읽기 전용 리소스를 권한으로 만들지 않습니다. 승격된 커넥터나 접점은 현재 범위와 관련 현재 담당 문서가 다시 담당해야 합니다.

## Category 경계

이 카테고리는 "사용자나 에이전트가 나중에 어디서 어떻게 하네스와 상호작용할 수 있는가?"가 핵심인 후보를 담습니다. 로컬 운영자 명령, `doctor` 접점, 읽기 전용 리소스, 대시보드와 호스팅 접점, 더 넓은 커넥터 생태계, 접점 간 표시나 검증 접점이 여기에 들어갑니다.

런타임 보안 주장, 아티팩트 캡처 저장소, 정책 카탈로그, 팀 생명주기는 담당하지 않습니다. 향후 접점이 그런 영역에 의존하더라도 이 문서는 승격 전 접점 관점의 후보만 기록합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 향후 로컬 운영자 명령 묶음 | `harness connect`, `harness serve mcp`, `harness doctor`, `harness projection refresh`, `harness reconcile`, `harness recover`, `harness export`, `harness artifacts check`, `harness conformance run` 같은 향후 CLI 명령. |
| 운영자 준비 상태와 `doctor` 접점 | 로컬 운영을 위한 향후 준비 상태와 진단 접점. |
| 상태 보기 새로고침과 최신성 진단 | 상태 보기의 상태 변경을 활성화하지 않는 향후 새로고침과 최신성 가시성 접점. |
| 이후 읽기 전용 리소스 | `policy`, `evidence-manifest`, `surface`, `report`, `bundle`, `journey`, `design` 같은 향후 읽기 전용 리소스. |
| 대시보드와 호스팅 작업 흐름 | 향후 대시보드, 호스팅 작업 흐름, 시각화, 카드 접점. |
| 더 넓은 커넥터와 호스팅 런타임 | 향후 커넥터 마켓플레이스, 호스팅 UI, 호스팅 런타임, 원격 런타임 후보. |
| 접점 간 검증 | IDE, CLI, 채팅, MCP, 호스팅 접점 사이의 향후 검증 가시성. |
| 커넥터 적합성 생태계 | 적합성 모음 자체를 정의하지 않는 향후 커넥터 대상 적합성 주장과 마켓플레이스 신호. |

## 후보 상세

### 향후 로컬 운영자 명령 묶음

요약: 향후 로컬 명령 접점과 운영자 진입점을 담습니다.

상태: 이후 후보.

현재 아님: 현재 MVP에 포함되지 않으며 현재 요구사항을 만들지 않습니다.

승격 조건: 승격 때 현재 범위 문서 갱신, 관련 현재 담당 문서 갱신 또는 새 담당 문서 지정, 동작이 규범이 되면 적합성/점검 갱신이 필요합니다.

현재 담당 문서 영향: 승격 전에는 없습니다.

## 승격 규칙

승격은 이 파일만 고쳐서 이루어지지 않습니다. 후보가 현재 기능이 되려면 같은 문서 전용 작업 묶음에서 현재 범위와 관련 현재 담당 문서가 함께 갱신되어야 합니다.

승격할 동작의 현재 담당 문서가 없다면, 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 그 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기의 언급은 커넥터, 호스팅 런타임, 원격 서비스, 대시보드, UI, 로컬 명령, 읽기 전용 리소스, 접점 간 권한을 활성화하지 않습니다.
