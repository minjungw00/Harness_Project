# 이후 후보: 정책과 적합성

## 이 문서가 담당하는 것

이 문서는 정책 계열, 검증기, 관문, 면제, 심각도 기반 차단, 실행 가능한 fixture 계열, 적합성 진입점에 관한 비활성 이후 후보를 담당합니다. 후보를 이 문서에 모아 [이후 후보 색인](index.md)이 경로 안내와 짧은 요약으로 남을 수 있게 합니다.

여기의 모든 후보는 향후 가능성입니다. 후보 상세는 문서 원천 자료일 뿐이며 현재 검증기, 관문, 면제, fixture, 적합성 동작을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP 닫기 준비 상태 요구사항, 현재 검증기, 현재 차단 사유 범주, 현재 면제 경로, 실행 가능한 fixture 모음, 적합성 보고서, API 동작, 저장 효과, 보안 보장, 구현 준비 상태를 정의하지 않습니다.

또한 정책, QA, 검증, 수락, 면제, 잔여 위험 판단을 하나의 승인 경로로 합치지 않습니다. 어떤 승격에서도 이 경계는 유지되어야 합니다.

## Category 경계

이 카테고리는 "하네스가 나중에 어떤 규범 정책이나 점검을 정의할 수 있는가?"가 핵심인 후보를 담습니다. 설계 정책, 수동 QA 관문, 검증 관문, 검증기 ID, 심각도 의미, fixture 계열, 적합성 진입점, 커넥터 적합성 정책이 여기에 들어갑니다.

제품 커넥터 접점, 아티팩트 캡처 세부 동작, 다중 사용자 작업 흐름, 보증 통제는 담당하지 않습니다. 단, 후보가 정책이나 적합성으로 표현되는 경우 이 문서가 승격 전 정책과 적합성 관점을 담당합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 수동 QA 작업 흐름과 `qa_gate` | 향후 수동 QA 관문과 관련 정책 후보. |
| 수동 QA 면제 `qa_waiver` | 사용자 소유 판단을 대체하지 않는 향후 수동 QA 정책 면제 경로. |
| 검증 관문 `verification_gate` | 향후 검증 관문 정책과 닫기 준비 상태 관계. |
| 설계 관문과 정책 차단 사유 | 향후 `design_gate`, `design_policy`, 차단 사유 범주, 설계 품질 정책. |
| 넓은 설계 검증기와 심각도 기반 차단 | 향후 검증기 ID, 심각도 의미, 차단 정책. |
| 전체 설계 품질 정책 계열 | `shared_design`, `domain_language`, `vertical_slice`, `feedback_loop`, `tdd_trace`, `deep_module_interface`, `codebase_stewardship` 같은 향후 정책 계열. |
| `ValidatorResult` 안정 ID와 정책 계열 | 향후 안정 검증기 식별, 정책 계열, 심각도, 면제 어휘. |
| 향후 fixture와 적합성 진입점 | 향후 fixture 계열, 실행 가능한 모음 진입점, 보고서, 적합성 실행 동작. |
| 고급 검증기와 인터페이스 확인 | 향후 고급 설계, 언어, 인터페이스 확인 검증기. |

## 승격 규칙

승격은 이 파일만 고쳐서 이루어지지 않습니다. 후보가 현재 기능이 되려면 같은 문서 전용 작업 묶음에서 현재 범위와 관련 현재 담당 문서가 함께 갱신되어야 합니다.

승격할 동작의 현재 담당 문서가 없다면, 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 그 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기의 언급은 관문, 차단 사유, 검증기, 면제, fixture 모음, 적합성 실행기, 보고서, 심각도 모델, 닫기 준비 상태 요구사항을 활성화하지 않습니다.
