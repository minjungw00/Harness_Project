# 작성 가이드

하네스 문서를 고치기 전에 이 작성 가이드를 사용합니다. 이 문서는 문서 편집 규칙일 뿐입니다. 하네스 서버/런타임 구현, 제품 저장소 쓰기, 생성된 운영 파일, 런타임 상태, 상태 보기, 증거 기록, QA 기록, 수락 기록, 닫기 기록, 잔여 위험 기록, 실행 가능한 fixture, 적합성 실행기를 승인하지 않습니다.

이 저장소는 현재 문서 전용입니다. [MVP 계획](../build/mvp-plan.md)의 유지보수자 인계 담당 문서가 다르게 말하지 않는 한 문서 검토 상태로 봅니다. 문서는 향후 하네스 서버를 위한 원천 자료이지, 수락된 구현 준비 상태나 구현된 런타임 동작이 아닙니다.

## 1. 읽기 순서와 범위

- 이 저장소에서 작업하기 전에 루트 `AGENTS.md`를 읽습니다.
- 한국어 문서를 편집하기 전에는 이 작성 가이드를 읽습니다.
- 이중 언어 편집이나 용어에 영향을 주는 편집이면 [번역 가이드](translation-guide.md)를 읽습니다.
- 영어 문서 의미도 바꾸면 [영어 작성 가이드](../../en/maintain/authoring-guide.md)와 [영어 번역 가이드](../../en/maintain/translation-guide.md)를 함께 봅니다.
- 작업은 문서 전용으로 유지합니다.
- 작은 작업 묶음을 선호합니다. 변경한 파일과 삭제한 파일을 보고합니다.
- 사용자가 명시적으로 요청하지 않으면 커밋을 만들지 않습니다.

오래된 문구가 현재 제품 명제, 담당 문서 경계, 한국어 품질 규칙, active/later 경계, 정직한 보안 표현과 충돌하면 다시 씁니다. 오래된 섹션 모양이 아니라 오래 남는 규칙을 보존합니다.

## 2. 경로 문서는 경로만 담당

README, Start, Use, Build, Maintain, 색인 문서는 독자를 안내합니다. 기술 계약의 두 번째 기준 문서가 되면 안 됩니다.

탐색에는 아래 경로 묶음을 사용합니다.

- `docs/doc-index.yaml`
- `docs/*/start.md`
- `docs/*/use/user-guide.md`
- `docs/*/use/agent-guide.md`
- `docs/*/use/judgment-examples.md`
- `docs/*/use/surface-recipes.md`
- `docs/*/build/mvp-plan.md`
- `docs/*/reference/README.md`
- `docs/*/later/index.md`
- `docs/*/maintain/authoring-guide.md`
- `docs/*/maintain/translation-guide.md`
- `docs/*/maintain/checks.md`

정확한 계약 담당 문서는 [참조 색인](../reference/README.md)에서 고릅니다. 승격되지 않은 이후 후보는 [이후 후보 색인](../later/index.md)에서 봅니다. [doc-index.yaml](../../doc-index.yaml)은 문서 검색 경로 정보일 뿐입니다. 런타임 설정, 구현 상태, 하나의 `doc_id`에 두 언어를 함께 싣는 허가가 아닙니다.

## 3. 계약 하나에는 담당 문서 하나

모든 엄격한 계약에는 기준 담당 문서가 하나만 있습니다. 정확한 필드, enum 값, DDL, 스키마, 알고리즘, 상태 전이, gate 규칙, fixture 본문 형태, 템플릿 본문, 저장소 규칙, 보안 보장, 오류 우선순위, 공식 정의는 [참조 색인](../reference/README.md)이 지정한 담당 문서에 둡니다.

기술 계약 내용을 추가하거나 바꿀 때는 이렇게 합니다.

- 규범 내용은 먼저 기준 담당 문서에 넣습니다.
- 의미가 바뀌면 영어와 한국어 담당 문서를 같은 작업 묶음에서 고칩니다.
- 담당 문서가 아닌 곳의 중복은 독자에게 보이는 짧은 결과와 담당 문서 경로로 바꿉니다.
- README, Maintain, 경로 문서, 예시는 스키마 표, DDL, 저장 효과, 공개 오류 우선순위, 보안 보장 정의, 템플릿 본문을 복사하지 않습니다.

Maintain 문서는 작성 규칙과 문서 점검을 담당합니다. API, 저장소, 보안, 상태 보기, 런타임, 제품 계약을 담당하지 않습니다.

## 4. active/later 경계

활성 문서는 이후 후보를 현재 MVP 요구사항처럼 보이게 만들면 안 됩니다. 값, 메서드, 테이블, fixture 계열, 명령, 템플릿, 커넥터 동작, 보안 보장은 기준 담당 문서가 범위, 대체 동작, 증명 기대와 함께 승격할 때만 활성입니다.

profile-gated 값을 기본 현재 MVP 값처럼 나열하지 않습니다. 이후 후보를 활성 요구사항처럼 설명하지 않습니다. 예시에 이후 후보가 필요하면 이후 전용이라고 표시하고 규범 세부사항은 [이후 후보 색인](../later/index.md)이나 그 후보를 승격한 담당 문서로 보냅니다.

## 5. 사용자 판단 경계

하네스는 사용자 소유 판단을 보존합니다. 제품 동작, 중요한 기술 방향, 범위 확장, 민감 동작 승인, 최종 수락, 잔여 위험 수락, 취소 판단은 서로 구분합니다. 넓은 승인은 특정 판단을 대신하지 않습니다.

최종 수락과 잔여 위험 수락은 빠진 증거를 만들거나 증거 공백을 지우지 않습니다. 민감 동작 승인은 제품 파일 쓰기 범위나 Write Authorization이 아닙니다.

## 6. 보안 표현

보안 표현은 문서화된 보장 수준과 맞아야 합니다.

- 하네스가 행동을 안내하거나 기록할 수 있지만 기술적으로 막지 못할 때는 협력형 표현을 씁니다.
- 행동 뒤 관찰 가능한 사실을 감지하거나 보고할 수 있고 해당 범위의 역량 확인이 통과했을 때만 탐지형 표현을 씁니다.
- 예방형 또는 격리 표현은 정확한 메커니즘과 증명 경로가 담당 문서에 있을 때만 씁니다.

보안 담당 문서가 정확한 메커니즘을 문서화하고 증명하지 않았다면 OS 권한, 임의 도구 샌드박스, 변조 방지 파일, 기본 도구 실행 전 차단, 보안 격리를 암시하지 않습니다.

## 7. 한국어 품질

한국어 문서는 줄 단위 번역이 아니라 자연스러운 한국어 기술 문장이어야 합니다. 파일 경로, `doc_id` 값, API 메서드 이름, 스키마 필드, enum 값, 오류 코드, 테이블 이름, validator ID, 템플릿 이름 같은 정확한 식별자는 그대로 보존합니다.

[번역 가이드](translation-guide.md)의 용어를 따릅니다. 어울리는 곳에는 "한영 문서 동시 유지", "의미 일치", "줄 단위 번역 아님", "에이전트 중복 주입 금지", "현재 MVP", "담당 문서", "사용자 소유 판단", "최종 수락", "잔여 위험 수락", "협력형 보장", "탐지형 보장", "profile-gated 값"을 사용합니다.

## 8. 오래된 내용

유지보수 문서는 앞으로의 편집을 안내해야 합니다. 과거 재작성 리뷰, 해결된 이슈 기록, 오래된 수락 기록, 오래된 별칭 이력, 과거 번역 문제 기록, 임시 마이그레이션 계획을 보존하지 않습니다.

아래처럼 처리합니다.

| 분류 | 처리 |
|---|---|
| `preserve` | 담당 문서에 맞는 유용한 의미를 보존합니다. |
| `shrink` | 독자에게 보이는 결과와 담당 문서 경로만 남깁니다. |
| `move` | 의미를 담당 문서로 옮기고 중복을 제거합니다. |
| `delete` | 오래되었거나, 오해를 만들거나, 과거 기록이거나, 중복인 문구를 삭제합니다. |
| `decision-needed` | 구현 준비 결정은 [MVP 계획](../build/mvp-plan.md)으로, 계약 결정은 담당 문서로 보냅니다. |

## 9. 사후 편집 체크리스트

- [ ] 편집이 문서 전용으로 남아 있습니다.
- [ ] 의미가 바뀐 편집은 영어와 한국어에 함께 반영되었습니다.
- [ ] 한국어 문장은 자연스럽고 정확한 식별자를 보존합니다.
- [ ] 엄격한 계약은 기준 담당 문서 하나에만 남아 있습니다.
- [ ] README, Maintain, 경로 문서, 예시는 기술 계약을 정의하지 않고 요약합니다.
- [ ] active/later 경계와 profile-gated 값이 분명합니다.
- [ ] 사용자 소유 판단 경로가 서로 섞이지 않습니다.
- [ ] 보안 표현이 문서화된 보장 수준과 맞습니다.
- [ ] 링크와 대응 언어 경로가 정상입니다.
- [ ] 임시 마이그레이션 메모, 임시 파일, 생성된 런타임 기록, 보관 사본이 남아 있지 않습니다.
- [ ] 관련 [문서 점검](checks.md)을 실행했거나 실행하지 못한 점검을 보고했습니다.
