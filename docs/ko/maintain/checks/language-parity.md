# 언어 일치 점검

이 점검은 한영 대응 문서의 의미가 변경되거나, 한국어 산문을 고치거나, 언어 사이에서 정확히 보존해야 하는 식별자를 다룰 때 사용합니다. 문서 품질 점검일 뿐이며, 어느 한 언어도 다른 언어의 보관본이나 부록이 아니고 제품 런타임 동작을 인증하지도 않습니다.

제목이 맞는지는 탐색 단서일 뿐입니다. 제목 일치를 통과해도 문단, 표, 목록, 예시, 주의 사항, 예외, 제거된 개념의 의미까지 일치한다는 뜻은 아닙니다. 영어와 한국어가 서로 맞아도 양쪽이 같은 잘못된 의미 라벨이나 구조를 공유한다면 충분하지 않습니다.

## CHK-PARITY-001: 영어와 한국어 의미 일치

점검 기준 문서:
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [한국어 번역 가이드](../translation-guide.md)
- [doc-index.yaml](../../../doc-index.yaml)

확인할 근거:
- 의미 변경이 있는 편집에서는 대응 파일을 의미 단위로 비교합니다.
- 대응 파일의 독자 목적, 규범 강도, 담당 문서 경로, 기준 범위/지원 범위 밖 경계, 사용자 판단 경계, 보안 보장 수준이 같은지 확인합니다.
- 제목이 맞는지만으로 충분하다고 보지 말고, 제목 아래 의미 단위를 직접 확인합니다.
- 한국어는 줄 단위 번역이 아니라 자연스러운 구조를 허용합니다.

실패 조건:
- 한 언어가 의미 변경을 빠뜨립니다.
- 한 언어가 대응 파일보다 규칙을 강하게 하거나 약하게 하거나 다른 담당 문서로 보냅니다.

수정 방향:
- 같은 문서 변경 범위에서 두 언어를 함께 고칩니다.
- 의미 일치를 유지하면서 한국어는 자연스럽게 다시 씁니다.

## CHK-PARITY-002: 정확한 식별자 보존

점검 기준 문서:
- [용어 지도](../../../terminology-map.yaml)
- [번역 가이드](../translation-guide.md)

확인할 근거:
- 정확한 식별자가 두 언어에서 그대로 남아 있는지 확인합니다.
- 파일 경로, 앵커, `doc_id` 값, API 메서드, 스키마 필드, enum 값, 테이블 이름, validator ID, 오류 코드, 제품 라벨은 문맥상 명확성이나 검색성이 필요할 때 백틱으로 감쌉니다.

실패 조건:
- 정확한 식별자를 번역, 지역화, 재서식화하거나 의미가 달라지는 표시 라벨처럼 씁니다.

수정 방향:
- 정확한 식별자를 복원합니다.
- 필요하면 식별자 옆에 평이한 설명을 덧붙입니다.

## CHK-PARITY-003: 한국어 구조 보존

점검 기준 문서:
- [한국어 번역 가이드](../translation-guide.md)
- [한국어 작성 가이드](../authoring-guide.md)
- [영어 작성 가이드](../../../en/maintain/authoring-guide.md)

확인할 근거:
- 한국어 참조 문서를 고쳤다면 조건, 결과, 예외, 비주장, 담당 문서 링크, 닫기 준비 상태의 결과를 의미 단위로 비교합니다.
- 한국어 문장의 줄 수와 순서는 달라도 되지만, 중요한 주의 사항과 담당 경계가 보이는지 확인합니다.
- 촘촘한 한국어 문단이 조건, 예외, 비주장을 합쳐 숨기지 않았는지 확인합니다.

실패 조건:
- 한국어 문구가 넓은 주제는 보존하지만 조건/결과/예외나 비주장 구조를 하나로 뭉갭니다.
- 담당 경계, 기준 범위/지원 범위 밖 경계, 보안 비주장, 닫기 준비 상태의 결과가 대응 의미 단위보다 찾기 어려워집니다.

수정 방향:
- 의미 단위가 드러나도록 자연스러운 한국어 문단이나 글머리표로 나눕니다.
- 정확한 식별자를 그대로 두고, 줄 단위 번역을 강제하지 않으면서 의미 일치를 보존합니다.

## CHK-PARITY-004: 한국어 저장소 구조

점검 기준 문서:
- [참조 색인](../../reference/README.md)과 `doc-index.yaml`에서 고른 적용되는 대응 저장소 담당 문서
- [번역 가이드](../translation-guide.md)

확인할 근거:
- 한국어 저장소 참조 문서를 고쳤다면 대응 영어 저장소 원천 문서를 의미 단위로 비교합니다.
- 조건, 효과, 예외, 비주장, 담당 문서 링크가 한국어에서도 분리되어 보이는지 확인합니다.
- 촘촘한 한국어 문단이 저장소 규칙을 합쳐 조건, 예외, 비주장을 숨기지 않는지 확인합니다.

실패 조건:
- 중요한 저장소 조건, 효과, 예외, 비주장이 촘촘한 한국어 문단 안에 뭉쳐 있습니다.
- 넓은 주제는 맞지만 저장소 경계가 대응 영어 의미 단위보다 검토하기 어려워집니다.

수정 방향:
- 의미 단위가 드러나도록 자연스러운 문단, 글머리표, 표로 다시 씁니다.
- 정확한 식별자는 그대로 두고, 유지보수 지침에 계약 세부사항을 반복하지 말고 저장소 담당 문서로 연결합니다.

## CHK-PARITY-005: 한국어 사용자 문서 가독성

점검 기준 문서:
- [한국어 작성 가이드](../authoring-guide.md)
- [한국어 번역 가이드](../translation-guide.md)

확인할 근거:
- 한국어 사용자 문서가 자연스러운 한국어 기술 문서인지, 한국어 개념을 먼저 두는지, 용어가 일관되는지 확인합니다.
- 정확한 식별자는 검색 가능하게 보존하되 일반 표시 라벨처럼 드러내지 않는지 확인합니다.
- 일반 영어 명사와 명사구는 정확한 식별자, 의도적 제품 라벨, 자연스러운 기술 차용어가 아니라면 한국어로 옮겼는지 확인합니다.
- 영어 어순을 그대로 비추는 것보다 자연스러운 한국어 구조가 같은 의미를 더 분명히 전달한다면 한국어 구조로 다시 썼는지 확인합니다.

실패 조건:
- 한국어 문장이 영어 어순을 비추거나, 피할 수 있는 영어 명사구를 남기거나, 내부 식별자 뒤에 독자 행동을 숨깁니다.
- 대응 영어 문장에 있었다는 이유만으로 일반 영어 명사를 한국어 산문에 남깁니다.
- 식별자나 의도적 라벨이 아닌 영어 명사 묶음을 화면에 보이는 산문으로 그대로 남깁니다.

수정 방향:
- 식별자와 의미 일치를 보존하면서 자연스러운 한국어로 다시 씁니다.

## CHK-PARITY-006: 대응 제목과 읽기 구조

점검 기준 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [작성 가이드](../authoring-guide.md)

확인할 근거:
- 제목 일치는 비교의 출발점일 뿐이며, 그 자체가 의미 일치 검토를 대신하지 않습니다.
- 제목이 바뀌면 대응 영어/한국어 파일을 제목 의미와 읽기 구조 기준으로 비교합니다.
- 섹션을 추가, 삭제, 분리, 병합했다면 대응 파일이 같은 의미 단위를 대응 제목이나 가까운 문맥으로 드러내는지 확인합니다.
- 두 언어가 같은 주요 의미 단위, 담당 문서 경로, 주의 사항, 예외, 체크리스트 범위를 드러내는지 확인합니다.
- 대응 제목 아래에서 표, 목록, 예외, 비주장, 담당 경계 규칙이 빠지거나 달라지지 않았는지 확인합니다.
- 한국어의 보이는 제목은 자연스럽게 두고, 안정적인 영어 앵커가 필요하면 숨김 앵커를 쓰는지 확인합니다.

실패 조건:
- 한 언어의 제목 변경이 대응 파일보다 의미 단위를 추가하거나, 제거하거나, 약하게 하거나, 다른 담당 문서로 보냅니다.
- 섹션 구조 때문에 한 언어에서는 규칙을 쉽게 찾을 수 있지만 다른 언어에서는 의미 있게 찾기 어려워집니다.
- 앵커를 맞추려고 한국어 제목이 영어 표현이나 어순을 보존합니다.

수정 방향:
- 같은 문서 변경 범위에서 대응 제목과 주변 맥락을 함께 갱신합니다.
- 안정적인 앵커가 필요하면 숨김 앵커로 보존하고, 보이는 한국어 제목은 자연스럽게 둡니다.

## CHK-PARITY-007: 표 의미 일치

점검 기준 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [작성 가이드](../authoring-guide.md)

확인할 근거:
- 변경된 섹션 안에서 대응 파일의 표 개수를 비교합니다.
- 대응 표의 머리글이 같은 의미를 가지며, 정확한 식별자가 그대로 보존되었는지 확인합니다.
- 행 수만 보지 말고 행의 의미를 비교합니다. 행을 나누거나 합칠 수는 있지만 조건, 값, 담당 문서 경로, 예외, 비주장이 모두 같은 의미 단위로 남아 있어야 합니다.
- 표가 섹션 안에서 놓인 위치가 같아서 같은 주제, 범위, 담당 경계에 적용되는지 확인합니다.

실패 조건:
- 한 언어에서 표를 추가, 삭제, 이동해 규칙, 매핑, 담당 문서 경로, 예외, 주의 사항이 다른 섹션에 적용됩니다.
- 표 머리글이 다른 개념으로 옮겨졌거나, 정확한 식별자를 빠뜨리거나, 행을 읽는 의미를 바꿉니다.
- 행의 의미가 추가, 삭제, 약화, 강화되거나 조건과 예외가 보존되지 않은 넓은 행으로 뭉쳐집니다.

수정 방향:
- 같은 문서 변경 범위에서 대응 표를 함께 고칩니다.
- 같은 의미 단위가 검토 가능한 상태로 남을 때만 행을 나누거나 합칩니다.
- 표를 대응 섹션으로 옮기거나, 가까운 문맥에 같은 범위와 담당 경계를 보존하는 문장을 둡니다.

## CHK-PARITY-008: 목록 의미 일치

점검 기준 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [작성 가이드](../authoring-guide.md)

확인할 근거:
- 규범 목록, 허용/금지 절, 의미하지 않는다는 절, 예외, 담당 경계 목록을 의미 단위로 비교합니다.
- 목록 순서를 바꿨다면 우선순위, 평가 순서, 경로 순서, 독자에게 보이는 결과가 달라지지 않았는지 확인합니다.
- 한국어 문장은 자연스러운 흐름을 쓸 수 있지만, 규칙, 예외, 비주장, 담당 링크를 담은 목록 항목은 빠짐없이 보존합니다.

실패 조건:
- 대응 목록의 길이는 같지만 규칙 의미가 다릅니다.
- 주장을 제한하거나, 예외를 이름 붙이거나, 담당 문서로 보내는 항목이 빠졌거나 넓은 문단에 흡수됩니다.
- 허용/금지 표현, 의미하지 않는다는 표현, 담당 경계 표현이 한 언어에서 더 강하거나 약하거나 넓어집니다.

수정 방향:
- 빠진 의미 단위를 목록 항목이나 가까운 이름 있는 블록으로 복원합니다.
- 규범 강도와 담당 문서 경로를 맞추면서 대응 목록을 자연스럽게 다시 씁니다.

## CHK-PARITY-009: 제거된 개념의 번역 잔여물

점검 기준 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [용어 지도](../../../terminology-map.yaml)

확인할 근거:
- 한 언어에서 개념 라벨, 담당 문서 경로, 값 이름, 표시 문구, 예시 용어를 제거했다면 대응 언어에서 정확한 문자열, 바꿔 쓴 표현, 번역 표현, 혼합어 변형을 찾습니다.
- 제거된 영어 라벨이 한국어 바꿔 쓰기, 번역, 표 행, 목록 항목, 제목, 용어집, 메타데이터, 표시 문구를 통해 남지 않았는지 확인합니다.
- 제거된 표현은 유지보수 문서나 용어 담당 문서가 검색 가능한 금지 표현으로 의도적으로 남길 때만 보존합니다.
- 제거된 표현을 검색 패턴으로 남긴다면 주변 문구가 그 문자열을 현재 문서 문장이 아니라 검토용 패턴으로 분명히 설명하는지 확인합니다.

실패 조건:
- 영어 라벨은 문자 그대로 사라졌지만 대응 한국어 문구가 같은 개념을 의미로 보존합니다.
- 표, 목록, 제목, 용어집 항목에 번역 잔여물이 남아 제거된 개념을 현재 쓰는 개념, 지원되는 개념, 담당 문서 경로처럼 보이게 합니다.

수정 방향:
- 잔여물을 제거하거나 안정적인 범주와 적용되는 담당 문서 링크로 바꿉니다.
- 유지보수자가 검색해야 하는 표현이라면 용어 담당 문서나 집중 유지보수 지침에서 금지 표현으로만 보존합니다.

## CHK-PARITY-010: 부정 절의 강도와 위치

점검 기준 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [작성 가이드](../authoring-guide.md)

확인할 근거:
- 변경된 대응 섹션에서 부정 절 단서를 찾습니다. 영어에서는 `Not allowed`, `Does not imply`, `Not implied`, `must not`을, 한국어에서는 `허용되지 않는 것`, `의미하지 않는 것`, `해서는 안 됩니다`를 확인합니다.
- 조건부 금지 단서도 함께 찾습니다. 영어에서는 `unless`, `only when`, `except when`을, 한국어에서는 `단`, `일 때만`, `경우를 제외하고`처럼 조건이나 예외를 드러내는 표현을 확인합니다.
- 금지, 예외, 비주장을 의미 단위로 비교합니다.
- 조건부 금지가 어떤 내용을 권한 근거로 취급할 수 있는 조건을 담고 있다면 한 언어에서 그 조건이 금지 라벨 안에 숨고 다른 언어에서는 조건이나 담당 경계로 드러나지 않는지 확인합니다.
- 한 언어가 다른 언어보다 더 강한 금지, 더 넓은 예외, 더 좁은 예외, 더 강한 비주장을 만들지 않는지 확인합니다.
- 한 언어가 금지나 비주장을 표, 목록, 이름 있는 블록, 가까운 문장에 두었다면 대응 언어도 그 의미 단위를 대응 위치에 두는지 확인합니다.

실패 조건:
- 부정 절이 한 언어에만 있어 독자가 피해야 하는 범위가 달라집니다.
- 한 언어가 주의를 금지로 바꾸거나 좁은 예외를 넓은 예외로 바꾸는 등 규범 강도를 키웁니다.
- 한 언어는 조건과 금지를 나누어 쓰지만 대응 언어는 그 조건을 `Not allowed` 또는 `허용되지 않는 것` 아래에 묻어 둡니다.
- 어떤 내용을 권한 근거로 취급할 수 있는 조건이 한 언어에서는 찾기 쉽지만 다른 언어에서는 금지 글머리표 안에 숨어 있습니다.
- 한 언어에서는 금지가 잘 보이는 표나 목록에 있지만 대응 언어에서는 다른 범위로 옮겨지거나, 촘촘한 문단에 묻히거나, 빠져 있습니다.

수정 방향:
- 빠진 금지, 예외, 비주장을 대응 언어에 복원합니다.
- 한국어 문장은 자연스럽게 쓰되 의미 단위의 규범 강도와 위치를 맞춥니다.
- 불명확한 조건부 금지는 양쪽 언어에서 조건, 금지된 행동, 담당 경계가 대응되도록 나눕니다.
- 부정 규칙이 제품 계약을 정의한다면 담당 문서로 옮기고, 양쪽 비담당 문서에는 짧은 경계와 담당 문서 링크만 남깁니다.

## CHK-PARITY-011: 의미 골격 일치

점검 기준 문서:
- [작성 가이드](../authoring-guide.md)
- [영어 작성 가이드](../../../en/maintain/authoring-guide.md)
- [번역 가이드](../translation-guide.md)

확인할 근거:
- 변경된 대응 섹션이 의미 골격을 사용하거나 암시한다면 산문을 비교하기 전에 의미 단위 골격을 확인합니다. 중요한 참조 섹션은 본문을 쓰거나 다시 다듬기 전에 골격을 먼저 확인합니다.
- 흔히 쓰는 골격에는 `Purpose`, `Conditions`, `Result`, `Non-claim`, `Owner boundary`, `Related references`가 있고, `Meaning`, `Contract`, `Boundary`, `Related references`도 사용할 수 있습니다.
- 조건부 금지 문구를 한 문장으로 두면 불명확하다면 양쪽 언어에서 `Conditions`, `Not allowed`, `Owner boundary`에 해당하는 같은 의미 골격을 사용합니다.
- 같은 섹션의 영어와 한국어가 같은 의미 골격을 쓰는지 확인합니다.
- 문장 수는 달라도 되지만 의미 단위의 위치와 규범 강도가 맞는지 확인합니다.
- 서로 맞는 골격도 그 라벨이 이름 붙이는 내용과 대조합니다. 의미 일치만으로 잘못 붙은 라벨이 올바르게 되지는 않습니다.
- 변경된 한국어 섹션에서 `조건`, `결과`, `비주장`, `허용되지 않는 것` 같은 구조 라벨을 찾고, 각 라벨에 대응하는 영어 의미 단위가 대응 섹션에 있는지 확인합니다.
- 한국어 문장이 자연스럽더라도 영어에 대응 의미 단위가 없으면 `조건`, `결과`, `비주장`, `허용되지 않는 것` 같은 라벨을 추가하지 않는지 확인합니다.
- 영어도 한국어에 대응 의미 단위가 없으면 `Not allowed`, `Does not imply`, `Non-claim` 섹션을 추가하지 않는지 확인합니다.

통과 기준:
- 대응 섹션이 같은 의미 단위 골격을 쓰고, 양쪽 언어에서 각 라벨이 그 라벨이 이름 붙이는 내용 유형과 맞습니다.

실패 조건:
- 대응 섹션의 제목이나 표는 맞지만 서로 다른 의미 골격을 사용합니다.
- 한 언어가 조건, 결과, 비주장, 담당 경계, 관련 참조 의미 단위를 추가하거나, 제거하거나, 다른 위치로 옮깁니다.
- 한 언어는 조건부 금지를 조건, 금지된 행동, 담당 경계로 드러내지만 다른 언어는 조건을 금지 단위 안에 숨깁니다.
- 두 언어가 같은 골격을 쓰지만 라벨이 내용 유형을 잘못 이름 붙입니다.
- 한국어가 영어 대응 단위 없이 `조건`, `결과`, `비주장`, `허용되지 않는 것`을 보이는 구조로 도입합니다.
- 한 언어에만 라벨이나 부정 섹션을 추가해 규범 강도를 바꾸거나 그 언어에서만 규칙을 더 쉽게 찾게 합니다.

수정 방향:
- 같은 섹션에 같은 의미 골격을 양쪽 언어에서 정하거나 복원합니다.
- 한국어 문장은 자연스럽게 다시 쓰되 의미 단위의 위치, 담당 문서 경로, 규범 강도를 보존합니다.
- 한쪽에만 있는 라벨을 제거하거나, 담당 문서가 뒷받침하는 의미라면 대응 의미 단위를 양쪽 언어에 둡니다.

## CHK-PARITY-012: 양쪽 언어가 같은 잘못된 의미 라벨을 공유하는 경우

점검 기준 문서:
- [작성 가이드](../authoring-guide.md)
- [구조 점검](structure.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [한국어 번역 가이드](../translation-guide.md)

확인할 근거:
- 대응 파일을 비교한 뒤, 각 언어에 대해 의미 라벨과 내용의 정합성 점검을 독립적으로 실행합니다.
- `Not allowed`, `Required behavior`, `Result`, `Does not imply`, `Owner boundary` 같은 라벨과 그에 대응하는 한국어 라벨이나 가까운 한국어 산문을 확인합니다.
- 의미 일치를 통과시키기 전에 양쪽 언어가 요구/금지, 효과/비효과, 경로/계약을 올바르게 분류했는지 확인합니다.
- `unless`, `only when`, `except when`에 해당하는 조건이 `Conditions`나 `Owner boundary`에 있어야 하는데 양쪽 문서가 모두 `Not allowed` 또는 `허용되지 않는 것` 아래에 둔 불명확한 조건부 금지를 공유하지 않는지 확인합니다.
- 한국어에서는 `허용되지 않는 것` 아래에서 `되어야 합니다` 또는 `해야 합니다`를 쓰는 문장을 확인합니다. 영어가 같은 금지 구조를 쓴다는 이유만으로 통과시키지 않습니다.
- 한국어 라벨은 영어 라벨을 비춘다는 이유만으로 통과시키지 않고, 영어 라벨도 한국어 문서와 구조가 같다는 이유만으로 통과시키지 않습니다.

통과 기준:
- 대응 언어가 의미 단위로 일치하고, 각 언어의 라벨과 구조도 그 내용 유형을 올바르게 설명합니다.

실패 조건:
- 영어와 한국어가 모두 요구되는 동작을 금지 라벨 아래에 두거나, 금지 행동을 요구사항 라벨 아래에 둡니다.
- 영어와 한국어가 모두 어떤 내용을 권한 근거로 취급할 수 있는 조건을 금지 글머리표나 행 안에 숨깁니다.
- 영어와 한국어가 모두 조건, 금지된 행동, 담당 경계로 나누어야 할 문장을 금지 라벨 아래에 둡니다.
- 영어와 한국어가 모두 효과를 비효과 라벨 아래에 두거나, 비효과를 결과 라벨 아래에 둡니다.
- 영어와 한국어가 모두 경로나 담당 경계 메모를 계약 본문처럼 제시합니다.
- 의미 일치 검토가 양쪽 언어가 맞는다는 이유로 통과하지만, 실제로는 양쪽이 같은 잘못된 의미 라벨이나 구조를 공유합니다.

수정 방향:
- 같은 문서 변경 범위에서 양쪽 언어의 의미 라벨이나 구조를 함께 고칩니다.
- 한국어는 자연스럽게 쓰되 고친 의미 단위, 담당 문서 경로, 규범 강도는 영어와 맞춥니다.
- 내용이 계약 세부사항이라면 담당 문서로 옮기고, 양쪽 비담당 문서에는 짧은 결과 설명과 담당 문서 링크만 남깁니다.
