# 에이전트 통합 참조

이 문서는 현재 문서 세트의 에이전트 커넥터 동작을 담당합니다. 접점별 사용법은 이 문서가 아니라 [접점별 사용법](../use/surface-recipes.md)이 담당합니다.

문서 원천 자료일 뿐입니다. 커넥터, MCP 서버, 런타임 상태, 생성된 manifest, 적합성 실행기를 구현하지 않습니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 커넥터 동작과 대체 동작
- 커넥터 경계에서의 `capability_profile` 의미
- 맥락 주입/가져오기 지침
- 에이전트 패킷 안의 확인된 접점 맥락
- 역량 확인에 따른 탐지형 표시 조건
- 에이전트 맥락에서 하나의 `doc_id`에 한 언어만 싣는 검색 지침

이 문서는 담당하지 않습니다.

- 접점별 사용법: [접점별 사용법](../use/surface-recipes.md)
- API 메서드나 스키마: [참조 색인](README.md)에서 고르는 API 담당 문서
- 저장소 배치나 아티팩트 생명주기: 저장소 담당 문서
- 보안 보장 의미: [보안](security.md)
- 정확한 템플릿 본문: [템플릿 본문](template-bodies.md)

## 커넥터 경계

커넥터는 하네스와 에이전트 접점 사이의 맥락을 전달합니다. 커넥터 설명, 생성 파일, 대화 텍스트, Product Repository 파일, 상태 보기, 에이전트 기억은 그 자체로 권한을 증명하지 않습니다. 로컬 접점 권한은 API와 보안 담당 문서가 정의한 등록되고 확인된 접점 맥락에 달려 있습니다.

Core, MCP, 상태 보기 데이터, 로컬 접근, 역량을 사용할 수 없거나 충분하지 않으면 커넥터는 그 제한을 드러내고 다음 안전한 행동을 관련 담당 문서로 보내야 합니다. 권한을 지어내면 안 됩니다.

## 맥락 규율

에이전트 맥락은 작게 유지합니다. 현재 작업 요약, 범위와 범위 밖, 대기 중인 사용자 소유 판단, 차단 사유, 다음 안전한 행동, 증거 공백, 닫기 준비 상태, 잔여 위험, 보장 수준, 출처 참조, 최신성 정도만 둡니다. 번역이나 의미 일치 검토가 필요한 경우가 아니라면 전체 스키마, DDL, 템플릿 본문, 과거 로그, 생성된 아티팩트, 같은 `doc_id`의 두 언어를 주입하지 않습니다.
