# Projection과 템플릿 참조

이 문서는 상태 보기 권한과 파생 표시 경계를 담당합니다. 정확한 템플릿 본문은 더 이상 이 문서가 담당하지 않고 [템플릿 본문](template-bodies.md)이 담당합니다.

문서 원천 자료일 뿐입니다. 런타임 상태 보기, 런타임 상태, 생성 산출물, 증거 기록, QA 기록, 최종 수락 기록, 잔여 위험 기록, 닫기 기록, 구현 준비가 끝난 서버 계획이 아닙니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 상태 보기 권한 경계
- 읽기 전용 파생 표시 규칙
- 상태 보기 출력의 최신성 및 사용할 수 없음 표현
- 렌더링된 라벨은 기준 스키마 값이 아니라 표시 문구라는 규칙
- 상태 보기 조정, 지속 저장되는 상태 보기 작업, 관리 블록 불일치 복구의 이후 전용 경계

이 문서는 담당하지 않습니다.

- 정확한 템플릿 본문: [템플릿 본문](template-bodies.md)
- 원천 Core 상태: [Core Model](core-model.md)
- 저장소 기록이나 상태 보기 저장소 후보: 저장소 담당 문서와 [이후 후보 색인](../later/index.md)
- 공개 API 스키마: API 스키마 담당 문서
- 커넥터 동작: [에이전트 통합](agent-integration.md)

## 권한 경계

Core가 소유한 상태와 지속 아티팩트 참조가 기준입니다. Projection, 상태 카드, Markdown 보고서, 렌더링된 템플릿, 대화 메시지, 커넥터 출력, 에이전트 맥락 패킷은 표시 또는 지원 맥락일 뿐입니다.

렌더링된 상태 보기, Markdown 상태 카드, 생성된 문서, managed block, front matter, 표시된 상태, 아티팩트 참조, 닫기 상태, 최종 수락 상태, 잔여 위험 상태, 템플릿 텍스트를 편집해도 Core 상태가 직접 바뀌지 않습니다.

## 파생 표시

상태 보기 출력은 향후 담당 문서가 지속 저장되는 상태 보기 작업을 승격하지 않는 한 읽는 시점에 현재 담당 기록에서 계산합니다. 범위, 증거 공백, 차단 사유, 최신성, 다음 안전한 행동, 잔여 위험, 현재 보장 표현을 사람이 읽게 돕습니다. 두 번째 상태 저장소가 아닙니다.

생성된 표시는 생략, 가림 처리, 차단된 아티팩트, 사용할 수 없음 메모를 보존해야 하며 숨긴 원천 값을 재구성하면 안 됩니다.

## 이후 경계

현재 MVP에는 활성 조정 큐, 편집 가능한 상태 보기 입력 경로, 상태 보기에서 Core로 되돌리는 복구 경로, 지속 저장되는 상태 보기 작업, 관리 블록 불일치 복구가 없습니다. 이런 항목은 범위, 대체 동작, 대체 불가능 규칙, 증명 기대와 함께 승격되기 전까지 이후 후보입니다.
