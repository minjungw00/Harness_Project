# 보안 참조

보안 문구, 로컬 접근 태세, 신뢰 경계, 보장 라벨을 정직하게 유지해야 할 때 이 문서를 사용합니다. 문서 원천 자료일 뿐이며 보안 제어를 구현하지 않습니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 보안 주장과 비보장
- 신뢰 경계 표현
- 협력형 보장과 탐지형 보장의 의미
- profile-gated 예방형 또는 격리형 표현 경계
- 민감 동작 승인과 제품 파일 쓰기 범위의 분리

이 문서는 담당하지 않습니다.

- API 메서드 동작이나 스키마: [참조 색인](README.md)의 API 담당 문서
- 저장소 배치, 아티팩트 생명주기, 잠금, 해시, 마이그레이션: [참조 색인](README.md)의 저장소 담당 문서
- 커넥터 동작이나 접점별 사용법: [에이전트 통합](agent-integration.md), [접점별 사용법](../use/surface-recipes.md)
- OS 수준 강제, 임의 도구 샌드박싱, 배포 보강

<a id="정직한-guarantee-display"></a>
## 현재 MVP 보장 수준

현재 MVP의 기본 표현은 협력형입니다. 연결된 접점이 절차를 따를 때 하네스가 담당 경로의 상태 변경을 안내하거나, 기록하거나, 비교하거나, 거절할 수 있다는 뜻입니다. 하드 차단, OS 권한, 임의 도구 샌드박싱, 변조 방지 강제, 격리가 아닙니다.

탐지형 표현은 대상 관찰 범위에 필요한 역량 확인이 통과했을 때만 쓸 수 있습니다. 복사된 `surface_id`, 생성 파일, 상태 보기, 대화 텍스트, Product Repository 파일, 에이전트 기억은 역량 증거가 아닙니다.

예방형과 격리형 주장은 향후 담당 문서가 정확한 메커니즘, 범위, 대체 동작, 증명 경로를 문서화하기 전까지 later/profile-gated 표현입니다.

## 명시적 비보장

현재 MVP는 아래 항목을 주장하지 않습니다.

- OS 수준 권한 제어
- 임의 도구 샌드박싱
- 변조 방지 로컬 파일 또는 저장소
- 보편적 도구 실행 전 차단
- 명령, 네트워크, 비밀값 관찰 기본 제공
- 명령, 네트워크, 비밀값 도구 실행 전 차단
- 보안 격리 또는 권한 격리
- 활성 보장으로서의 접점 자체 아티팩트 캡처

## 민감 동작 경계

민감 동작 승인은 이름 붙은 민감한 행동에 대한 사용자 소유 판단입니다. 제품 파일 쓰기 범위, Write Authorization, 최종 수락, 잔여 위험 수락, 증거, 아티팩트 권한, OS 권한, 도구 샌드박싱이 아닙니다.

제품 파일 쓰기 범위는 쓰기 경로의 Core/API/저장소 담당 문서가 다룹니다. 넓은 승인은 민감 동작 승인이나 제품 파일 쓰기 호환성을 대신하지 않습니다.
