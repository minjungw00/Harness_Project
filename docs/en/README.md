# Harness documentation

This is the English entry point for the Harness documentation set. It helps readers choose the right page; it does not define technical contracts.

## Start here

New readers should begin with [Start](start.md). It gives the first model for what Harness is for and how the rest of the documentation is organized.

## Read by role

- Working users should read the [User Guide](use/user-guide.md).
- Agents should read the [Agent Guide](use/agent-guide.md).
- Readers who need concrete judgment patterns should use [Judgment Examples](use/judgment-examples.md).

## Current scope

The current active MVP boundary is owned by [Active MVP Scope](reference/active-mvp-scope.md). Use that page for scope decisions instead of treating this README as a source of delivery detail.

## Reference contracts

Technical readers should start with the [Reference Index](reference/README.md). It routes each contract topic to its owner so this entry point can stay short.

## Maintenance and translation

Maintainers should use the [Authoring Guide](maintain/authoring-guide.md), [Checks](maintain/checks.md), and [Translation Guide](maintain/translation-guide.md). English and Korean pages should stay semantically aligned without requiring line-by-line translation.
